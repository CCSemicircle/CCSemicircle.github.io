1 数据结构和算法

1.1 数据结构的原则

  1. 数据结构：一类数据的表示及其相关的操作。
  2. 有效率(efficient)的算法：
    • 一个算法如果能在所要求的资源限制(resource constraint)内将问题解决好，则称这个算法是有效率的。
    • 一个算法如果比已知算法所需要的资源都要少，那么这个算法也可以称为是有效率的，而不管是否有其他特殊要求。
  3. **算法的代价(cost)**：一个算法所消耗的资源量。一般来说代价是由一个关键资源（如时间）来评估的，前提是这个算法满足其他资源限制。
  4. 选择数据结构的步骤（原则）：
    • 确定必须支持的操作
    • 衡量每种基本操作会遇到的资源限制
    • 选择最接近这些代价的数据结构

1.2 抽象数据类型和数据结构

  1. **类型(type)**：一组值的集合，如布尔类型由true和false组成。
  2. 简单类型和组合类型：
    • 简单类型：整数、浮点数等。
    • 聚合类型(aggregate type)和组合类型(composite type)：包含多项简单类型信息。
  3. **数据项(data item)**：一条信息或者其值属于某个类型的一条记录，数据项也可说成是数据类型的成员(member)。
  4. **数据类型(data type)**：一个类型和定义在这个类型上的一组操作。
  5. **抽象数据类型(abstract data type, ADT)**：数据结构作为一个软件构件的实现。ADT的接口利用一个类型和这个类型上的一组操作来定义，每一个操作有它的输入和输出定义。一个ADT并不指定数据类型是如何实现的，这些实现细节对于ADT的用户是隐藏的，并且通过封装(encapsulation)这个概念来组织外部对它的访问。
  6. **数据结构(data structing)**：ADT的实现。在如C++之类的面向对象语言中，ADT和它的实现共同组成了“类”(class)。此时，ADT的每一个操作均由一个成员函数(member function)或方法(method)来实现。定义数据项所需要的存储空间的变量称为“数据成员”(data member)。“对象”(object)是类的一个实例，即它在一个计算机程序的执行期间创建，并占用一些存储空间。
  7. 术语指代：
    • 数据结构：常指存储在计算机内存种的数据。
    • 与其相关的文件结构：长治外存储器(如磁盘驱动器、CD)种数据的组织。
  8. **数据项/数据类型有逻辑形式(logical form)和物理形式(physical form)**：
    • ADT定义了数据项/数据类型的逻辑形式
    • 数据结构实现了数据项/数据类型的物理形式。

1.3 设计模式

  • **设计模式(dasign pattern)**：重用设计模型来组合软构件。

1.3.1 享元模式(Flyweight)

如果在一个系统中存在多个相同的对象，那么只需要共享一份对象的拷贝,而不必为每一次使用都创建新的对象。

1.3.2 访问者模式(Visitor)

一种操作一组对象的方式，它的目的是不改变对象的定义，但允许新增不同的访问者，来定义新的(读取)操作。

1.3.3 组合模式(Composite)

用于把一组相似的对象当作一个单一的对象。依据树形结构来组合对象，用来表示部分以及整体层次。这种类型的设计模式属于结构型模式，它创建了对象组的树形结构。这种模式创建了一个包含自己对象组的类。该类提供了修改相同对象组的方式。

1.3.4 策略模式(Strategy)

封装一系列可替换的方法，这些可变的方法会在一个更大的行为种执行。如，排序函数接收的比较器(Comparator)后，无需知道数据项如何实现仍可进行比较排序。

1.4 问题、算法和程序

  1. **问题(problem)**：一个需要完成的任务，不包含有关怎样解决问题的限制。
  2. **算法(algorithm)**：解决问题的一种方法或一个过程。算法包含以下几条性质：
    • 正确性(correct)——正确解决问题
    • 具体步骤(concrete step)——具有完成算法的可执行步骤
    • 确定性(no ambiguity)——每一个步骤都是确定的
    • 有限性(finite)——具有有限步骤
    • 可终止性(terminate)——不能陷入死循环
  3. **程序(program)**：使用某种程序设计语言对算法的具体实现。

3 算法分析

3.1 概述

  1. 时间代价：影响时间代价的最主要因素一般是输入的规模，经常把执行算法所需要的时间T写成输入规模的函数，T(n)。当输入规模对运行时间不会产生影响，恒为某个值时，称为**常数运行时间(constant running time)**。
  2. **增长率(growth rate)**：指当输入的值增长时，算法代价的增长速率。
    • 线性增长率(linear growth rate)：cn
    • 二次增长率(quadratic growth rate)：n^2
    • 指数增长率(exponential growth rate)：2^n

3.2 渐进分析

3.2.1 上限

最差情况

  • 对于非负函数T(n)，如果存在两个正常数c和n0，对任意n > n0，有T(n) ≤ cf(n)，则称T(n)在O(f(n))中。

3.2.2 下限

最好情况

  • 若存在两个正常数c和n0，对于n > n0，有T(n) ≥ cg(n)，则称T(n)在集合Ω(g(n))中。

3.2.3 Θ表示法

平均情况

  • 如果一种算法既在O(h(n))中，又在Ω(h(n))中，则说平均情况下，它是Θ(f(n))。

3.2.4 化简法则

  1. 若f(n)在O(g(n))中，且g(n)在O(h(n))中，则f(n)在O(h(n))中；
  2. 若f(n)在O(kg(n))中，对于任意常数k > 0 成立，则f(n)在O(g(n))中；
  3. 若f1(n)在O(g1(n))中，且f2(n)在O(g2(n))中，则f1(n)+f2(n)在O(max(g1(n), g2(n)))中；
  4. 若f1(n)在O(g1(n))中，且f2(n)在O(g2(n))中，则f1(n)f2(n)在O(g1(n)g2(n))中。

3.2.5 函数分类

求n-> ∞, lim( f(n) / g(n))

  • 趋向于 ∞，f(n) 在Ω(g(n))中，因为f(n)增长得更快
  • 趋向于0，f(n)在O(g(n))中，因为g(n)增长得更快

3.2.6 空间代价

  • 所有并非真正数据得附加信息成为结构性开销(overhead)，如链表中每个元素附带指向下一个元素的指针。
  • 时间代价也可以用增长率衡量。
