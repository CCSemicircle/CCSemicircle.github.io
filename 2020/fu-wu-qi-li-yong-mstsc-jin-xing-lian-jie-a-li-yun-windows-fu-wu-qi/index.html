利用MSTSC进行连接阿里云Windows服务器

环境配置 MSTSC
服务器
发布日期:   2020-10-04
文章字数:   999
阅读时长:   3 分
阅读次数:  

1 配置规则

  1. 在实例详情里面进入“本实例安全组”界面，
  2. 进入对应实例界面，选择“快速创建规则”
  3. 选择常用的端口，如：SSH, HTTP, HTTPS, MYSQL等，
    授权对象自行选择：
    将端口授权给单个IP地址，填写：123.123.123.123，直接填IP地址即可；
    将端口授权给IP地址段，填写：10.10.10.1/24，直接填IP地址段；
    将端口授权给全部IP，填写：0.0.0.0/0，将端口授权给所有人。

    2 登录连接

    2.0 重置实例密码

    第一次登录需要重置实例密码，如图，选择“重置实例密码”。

    Tips：每次重置实例密码之后必须重启才能生效！

2.1 网页远程连接——不支持文件传输

网页远程连接服务器有两个途径，workbench 与 VNC。

2.1.1 workbench连接

workbench是直接远程连接桌面，如图，用户名在购买之后会以短信形式发送给你，一般是administrator。

2.1.2 VNC连接

VNC 是 virtual network connection的简称，第一次连接会给一个“远程连接密码”，只会出现一次，一定要记录下来。
Tips：“远程连接密码”，只会出现一次，一定要记录下来 ！

登录之后，在左上角输入指令。

选择 “CTRL + ALT + DELETE” 进入登录界面，密码是之前设置的“实例密码”，不是远程连接密码。

2.1.3 workbench 与 VNC 的区别

  1. 离开连接页面之后，workbench连接可以保持很久，VNC会很快关闭。

    2.2 利用MSTSC连接——Windows 系统 连接 Windows 系统

  3. MSTSC（Microsoft terminal services client），创建与终端服务器或其他远程计算机的连接，编辑现有“远程桌面连接(.rdp)”配置文件，并将 Windows XP 连接（使用“客户端国防部设备 接管理器”创建的连接）迁移到新的 .rdp 文件中。MSTSC还有一种说法，Microsoft Telnet Screen Control ，即“微软远程桌面控制”。
  4. 打开“控制面板” > “用户账户” > “凭据管理器” 界面 ，如图，选择“添加Windows凭据”。
  5. 如图输入相应信息，点击“确认”即可。
  6. 打开MSTSC连接界面：“win + R” 进入下图界面，输入 “mstsc”，点击“确认”即可。
  7. 经过上面的步骤进入下图界面会自动补全服务器网址，有多个网址时在下拉栏选择，如有其它网址未保存至本地计算机“Windows凭据”，需要手动输入服务器公网网址与用户名。
  8. 点击左下角的“显示选项”，进入如下界面
  9. 点击“本地资源”，进入如下界面
  10. 点击“详细信息”，勾选你要映射到服务器的磁盘
    Tips：经过映射的磁盘里的文件在连接服务器之后可直接复制到服务器上。
  11. 点击“连接”，出现如图警告，选择“查看证书”
  12. 选择“安装证书”
  13. 选择“本地计算机”，进入“下一步”
  14. 勾选“将所有证书都放入下列存储”，浏览选择“受信任的根证书颁发机构”，进入“下一步”
  15. 点击“完成”，出现此窗口即可
  16. 点击“确认”，然后下图也点击“确认”，
  17. 最后点击“是”，即完成连接。
  18. 之后再进行登录，会出现“名称不匹配”错误，选择“是”忽略即可。

    2.3 利用XShell和XFTP配合连接——适用于Linux、UNIX系统

PS：

  1. 利用XShell和XFTP也可连接Windows系统的服务器，但步骤繁琐，远不如MSTSC简单，有兴趣的朋友可以参考此文章：《Windows安装OpenSSH服务》。
  2. 利用XShell和SFTP连接Linux或UNIX服务器请跳转至本人另一篇博客《Linux入门》。

Tips：
1. 附XShell和XFTP的下载链接

  1. 官网下载：XSell，XFTP（需分开下载）
  2. 本人上传资源：含 XSHell 5 和 Xftp_5.0.543，详见本文附加资源。

2. 想要详细了解XShell 与 XFTP 可参考此文章：《Xshell和Xftp区别》
